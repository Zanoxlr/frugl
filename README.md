# Frugl

**Stop overpaying.**

A B2C "are you overpaying on your bills?" advisor. Run a free 60-second check, see a concrete number ("You're overpaying ~€340/year"), and switch to a better offer.

## Value proposition

> You're probably overpaying. See how much in 60 seconds.

- **Free check** — no registration, no bank access.
- **Concrete number** — surfaces the euro amount you're losing, per year.
- **Switch help** — we handle the move to a better provider.
- **Watchdog** — keeps your bills honest over time.

Guarantee (win-your-money-back): if we don't find you a saving, it costs you nothing.

## Money model

- Affiliate commission from the provider on switch.
- Watchdog membership (continuity → LTV).

CFA gate: free + affiliate works as a money model only if `(free→switch conversion × CPA) > CAC` within 30 days. Target CAC ≈ 0 via a viral share-card so the model works "for free."

## Why "Frugl"

Built from *frugal* — spending wisely, not wasting money. The dropped final vowel follows the modern tech-brand pattern (Flickr, Tumblr, Scribd), reads instantly as "an app," and is more brandable/defensible than a dictionary word. Five letters, one clean sound, travels across languages, works as a verb ("I frugled my bills this weekend").

Positioning note: frame it as smart, intentional saving ("stop overpaying"), not "be cheap."

Before committing: verify frugl.com / .io / .app availability and run EUIPO + USPTO + Google trademark checks.

## Build = validation

First build is also the cheapest test.

- **Week 1** — free-check landing + quiz. Tests whether the hook converts and the share-card spreads.
- **In parallel (a few days)** — Test 1: 5–10 real bills, old price vs best (Tarifnik/AKOS) → is the delta in telco/insurance >15–20%? Test 2: contact 3–5 providers → do they pay affiliate commission?
- **After ~2 weeks** — if landing converts + delta is large + affiliate rails exist → build switch fulfillment. Otherwise adjust (success-fee / different vertical).
